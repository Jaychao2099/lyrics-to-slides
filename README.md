# 歌曲投影片生成器

一款專為敬拜團隊設計的桌面應用程式，能夠輕鬆將歌曲轉換為簡易的投影片。

## 主要功能

- **歌詞自動搜尋**：輸入歌曲名稱，自動從網路搜尋歌詞，並支援手動編輯。
- **多種AI背景生成**：根據歌詞內容和氛圍，使用 AI 模型生成適合的投影片背景圖片（也可選擇本地圖片）。
- **Marp 投影片生成**：將歌詞和圖片自動排版成 Marp Markdown 格式，方便預覽和匯出。
- **投影片即時預覽**：編輯歌詞或調整設定時即時預覽最終效果。
- **多格式匯出**：支援 PDF、PPTX、HTML 等多種格式匯出。
- **批次管理功能**：創建歌曲集合，批次生成和匯出集合多首歌曲的投影片。
- **設定管理**：管理多種AI模型的 API 金鑰、圖片生成提示詞模板、自定義 Marp 標頭等。
- **本地快取**：歌詞、圖片、生成的投影片內容會儲存在本地快取中，加速載入。
- **本地數據庫**：使用SQLite儲存所有歌曲、集合和設定資料。
- **多語言支援**：目前僅支援**繁體中文**介面，預計未來支援英文與簡體中文。

## 安裝方式

### Windows 系統


1. 從[發布頁面](https://github.com/Jaychao2099/lyrics-to-slides/releases)下載最新的 `lyrics-to-slides-setup-1.1.2.exe` 安裝檔 或 `Lyrics to Slides-1.1.2.exe` 免安裝執行檔（可跳過2. 3.）。
2. 執行安裝檔，依照指示完成安裝。
3. 從開始選單或桌面快捷方式啟動應用程式。

### macOS 系統 (尚未支援)

*(注意：macOS 版本理論上可透過 `electron-builder` 建置，但可能未經完整測試)*

3. 將應用程式拖曳至應用程式資料夾。

## 快速開始

### 第一步：設定 API 金鑰

1. 開啟應用程式，點擊側邊欄的「設定」。
2. 前往「API 金鑰設定」分頁。
3. 輸入您想使用的AI模型API金鑰：
   - Google API金鑰
   - OpenAI API金鑰
   - Gemini API金鑰
   - Grok API金鑰
   - Anthropic API金鑰
4. 點擊「儲存設定」。

### 第二步：AI 功能設定
1. 前往「API 功能設定」分頁。
2. 分別為以下功能選擇 AI 模型，或選擇不使用 AI:
    - 歌詞搜尋
    - 提示詞生成
    - 圖片生成

### 第三步：搜尋歌詞

1. 點擊側邊欄的「搜尋歌詞」或「編輯歌詞」。
2. 輸入歌曲名稱和歌手/樂團（可選）。
3. 點擊「搜尋歌詞」，應用程式將嘗試從網路上爬取歌詞。
4. 確認搜尋結果，必要時可手動編輯歌詞。

    或

1. 從「已儲存歌曲」列表中選擇歌曲。

### 第四步：選擇背景圖片
* 點擊「生成背景圖片」，AI 將生成提示詞，並自動用提示詞生成圖片。

    或

* 點擊「匯入本地圖片」。

### 第五步：生成與編輯投影片

1. 確認歌詞和圖片後，系統會自動生成 Marp Markdown 格式的投影片內容。
2. 您可以在「編輯歌詞」分頁直接編輯 Markdown 內容進行微調，調整文字顏色、邊框等樣式。
3. 在「預覽投影片」分頁檢視效果。
4. 滿意後點擊「匯出投影片」，選擇您需要的格式（PDF, PPTX, HTML）。

### 第六步：批次處理（可選）

1. 點擊側邊欄的「批次投影片」。
2. 創建新的歌曲集合。
3. 從**已搜尋或儲存**的歌曲中添加多首歌曲至集合。
4. 拖曳歌曲調整順序後，點擊「生成批次投影片」。
5. 在「預覽」分頁檢視合併後的效果。
6. 點擊「匯出批次投影片」，選擇輸出格式和路徑進行匯出。

## 常見問題

### AI 功能設定建議怎麼選？
- 歌詞搜尋：**Gemini 系列** 或 **不使用AI (僅使用 Google 搜尋)**
- 提示詞生成：**Gemini 2.5 pro**
- 圖片生成：**OpenAI DALL-E-3** 或 **Gemini 2.0 Flash**
- Gemini 提供免費 api 使用額度，詳情可見[Gemini 官網](https://ai.google.dev/gemini-api/docs/rate-limits?hl=zh-tw)

### 無法搜尋到歌詞
- 檢查您的網路連接是否正常。
- 嘗試使用更精確的歌曲名稱和歌手名稱。
- 確認目標歌詞是否存在於公開的歌詞網站上，並且網站結構未發生重大變化。
- 若持續失敗，可嘗試手動複製貼上歌詞。

### 圖片生成失敗
- 確認您的API金鑰設定正確且有效、有足夠的額度。
- 每個AI模型可能有不同的錯誤原因，查看日誌獲取詳細資訊。
- 檢查網路連接。
- 嘗試修改「設定」中的「提示詞模板」，確保其符合相應AI服務的內容政策。
- 檢查歌詞內容是否觸發了內容審核。
- 嘗試切換到其他AI模型。

### 如何自訂投影片風格？
- 在「設定」頁面的「提示詞模板」分頁可自訂 AI 生成圖片的風格。
- 在編輯頁面的「源代碼與設置」分頁可直接修改 Markdown 內容調整排版，並設定文字大小與顏色、邊框大小與顏色。
- 支援上傳自定義背景圖片替代 AI 生成圖片。
- 在「設定」頁面的「文件模板」分頁可以自定義 Marp 標頭內容，進一步控制樣式。

### 一、啟用 Marp 模式

```markdown

marp: true

```
#### ✅ 說明：

可加入 `paginate: true` 顯示頁碼：
```markdown
---
marp: true
paginate: true
---
```

### 二、自訂整體樣式

```markdown
style: |
  section {
    text-align: center;
    font-size: 80px;
  }
```

#### ✅ 說明：
這段控制每張投影片（section）的樣式。

#### ⚙️ 可選項：
- `text-align:` 設定文字位置 (`left`, `center`, `right`)
- `font-size:` 設定整體字體大小。
- `background-color:` 設定背景顏色。
- `padding:` 增加字型間距。

### 三、標題樣式設定

```markdown
  h1 {
    position: absolute;
    top: 20px;
    right: 40px;
    font-size: 20px;
  }
```

#### ✅ 說明：
這段僅影響"標題（`h1`）"。

#### ⚙️ 可選項：
- `position: absolute;` 設定文字顯示於絕對位置。
- `font-size:` 設定文字大小
- `color:` 改變字體顏色
- `font-family: "Arial";` 指定字體

### 四、針對不同歌曲類別的樣式（使用 CSS 選擇器）

```markdown
section.song耶穌對我真好 p {
  color: #000000;
  text-shadow: ... #ffffff;
}
```

#### ✅ 說明：
針對 `.song耶穌對我真好` 這個類別的"一般內文（`p`）"與"標題（`h1`）"進行額外的樣式設定。

```markdown
section.song耶穌對我真好 p {
  color: #ffffff;
  text-shadow: ... #616161;
}
section.song耶穌對我真好 h1 {
    color: #000000;
}
```

- 可以為每首歌曲設定不同的視覺效果。

#### ⚙️ 可選項：
- 與前述相同。

## 資料與隱私

- 所有設定、搜尋到的歌詞、生成的圖片和投影片內容都儲存在您的**本地電腦** (`%APPDATA%/lyrics-to-slides` 或 macOS 對應的應用程式支援目錄下)，不會上傳至任何伺服器。
- API 金鑰使用 `electron-store` 儲存在本地設定檔中，建議確保您的電腦安全。
- 所有歌曲和集合數據儲存於本地SQLite數據庫中。
- 您可以在「設定」的「快取管理」分頁中清除所有快取資料（歌詞、圖片、投影片）。

## 專案架構

本專案使用以下技術堆疊:
- **Electron**: 跨平台桌面應用框架
- **React**: 用戶界面開發
- **TypeScript**: 靜態類型檢查
- **MUI (Material-UI)**: UI組件庫
- **SQLite**: 本地數據存儲
- **React DnD**: 拖放功能實現
- **Marp**: Markdown投影片框架

## 系統需求
